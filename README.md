# Pomelo: Современная платформа для поиска и бронирования туров 🍊

**Pomelo** — это дипломный проект, представляющий собой полнофункциональную веб-платформу для туристического бизнеса. Проект включает в себя клиентскую часть для пользователей, панель инструментов (Dashboard) для управления поездками и комплексную административную панель для менеджеров.

## 🚀 Стек технологий

*   **Backend**: Python 3.10+, FastAPI (асинхронный веб-фреймворк).
*   **Database**: PostgreSQL + SQLAlchemy (ORM).
*   **Migrations**: Alembic.
*   **Frontend**: React 19, TypeScript, Vite, CSS3 (Modern Flexbox/Grid).
*   **Design**: Авторская дизайн-система с поддержкой адаптивности и анимаций.
*   **Integrations**: Unsplash API (автогенерация контента).

## 🛠 Установка и запуск

Для работы проекта необходимы установленные **Python 3.10+** и **Node.js 18+**.

### 1. Подготовка базы данных
1.  Установите **PostgreSQL**.
2.  Создайте базу данных с названием `pomelo`.
3.  Настройте переменные окружения (см. ниже).

### 2. Настройка Backend
Перейдите в папку `backend`:
```bash
cd backend
```
Создайте виртуальное окружение и активируйте его:
```bash
python -m venv venv

venv\Scripts\activate

```
Установите зависимости:
```bash
pip install -r requirements.txt
```
Создайте файл `.env` в папке `backend` на основе следующего примера:
```env
DB_USER=postgres
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432
DB_NAME=pomelo
SECRET_KEY=your_secret_key_here
UNSPLASH_ACCESS_KEY=your_key_from_unsplash  # Для генерации фото
```
Примените миграции и заполните БД начальными данными:
```bash
alembic upgrade head
python seed.py --reset
```
Запустите сервер:
```bash
python main.py
```
Сервер будет доступен по адресу: `http://localhost:8000`

### 3. Настройка Frontend
Перейдите в папку `frontend`:
```bash
cd frontend
```
Установите зависимости:
```bash
npm install
```
Создайте файл `.env.local`:
```env
VITE_API_URL=http://localhost:8000
```
Запустите проект:
```bash
npm run dev -- --host 
```
Приложение откроется по адресу: `http://localhost:5173`
И в консоль будет выведен второй адрес для локальной сети, чтоб проверить работу на телефоне

## 📺 Описание разделов приложения

### 1. Главная страница (Home)
Витрина туров с ярким Hero-блоком, категориями (Пляж, Горы, Круизы и т.д.) и списком популярных предложений. Используется интеллектуальная загрузка изображений.

### 2. Поиск и фильтрация (Search)
Полноценный каталог с возможностью фильтрации по странам, категориям и датам. Все данные подгружаются динамически из API.

### 3. Детальная страница тура (Tour Details)
Подробное описание программы тура, перелета, проживания и питания. Включает фотогалерею, отзывы пользователей и виджет бронирования.

### 4. Личный кабинет (Dashboard)
Место, где пользователь управляет своим профилем, видит историю бронирований, список избранного и кошелек (баланс).

### 5. Панель администратора (Admin Panel)
Мощный инструмент для управления платформой:
*   **Dashboard**: Статистика продаж и активности.
*   **Туры**: Создание, редактирование и удаление туров.
*   **Категории**: Управление типами отдыха.
*   **Бронирования**: Обработка заказов (подтверждение/отмена).
*   **Настройки**: Управление комиссией системы и режимом обслуживания (Maintenance Mode).

## 💡 Обоснование принятых решений

Данный проект является **прототипом (MVP)** в рамках дипломной работы, поэтому некоторые системы были реализованы в упрощенном виде:

*   **Восстановление пароля (Hardcoded)**: Реальная отправка писем требует сложной настройки SMTP-серверов и доменной почты. Для демонстрации логики работы системы используется фиксированный код подтверждения (`1234`). Это позволяет протестировать весь UX-путь без внешних зависимостей.
*   **Пополнение баланса (Fake)**: Интеграция с реальными эквайрингами (Stripe/LiqPay) требует юридического лица. В проекте реализована имитация транзакции с валидацией карты (алгоритм Луна), чтобы показать работу кошелька и систему списаний.
*   **Автогенерация контента**: Чтобы проект выглядел профессионально и не требовал ручного добавления сотен фото, интегрировано **Unsplash API**. Фотографии подбираются по ключевым словам локации и категории тура.
*   **Кураторство локаций**: При создании тура страны и города выбираются из списка заранее определенных пар (например, Италия - Рим). Это гарантирует, что API для поиска фото выдаст качественный и релевантный результат, а также защищает систему от ошибок геопозиционирования.
*   **Система кэширования**: Для изображений (аватары, фото туров) реализована стратегия `No-Cache` на стороне сервера. Это критично для мгновенного обновления UI после загрузки новых файлов.
